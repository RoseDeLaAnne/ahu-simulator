# Prompt для русскоязычной и отечественной источниковой базы

Дата: 2026-05-02.

Этот prompt дополняет стартовый prompt из
`docs/37_3d_visualization_start_prompt.md`. Его задача - удержать проект в
российском нормативном, учебном и прикладном контексте: литература, исходные
данные, сценарии и пояснения должны опираться на отечественные источники, а не на
зарубежные справочники.

## Проверенные ориентиры источников

| Тип | Предпочтительный источник |
| --- | --- |
| Нормы ОВК | Росстандарт: `СП 60.13330.2020` "Отопление, вентиляция и кондиционирование воздуха" |
| Климатические данные | Росстандарт: `СП 131.13330.2025` "Строительная климатология" |
| Микроклимат помещений | Росстандарт: `ГОСТ 30494-2011` "Здания жилые и общественные. Параметры микроклимата в помещениях" |
| Санитарные требования | Официальный портал правовой информации / Роспотребнадзор: `СанПиН 1.2.3685-21` |
| Научные и отраслевые публикации | НЭБ/РГБ, ИСТИНА МГУ, eLIBRARY, КиберЛенинка, журнал АВОК |
| Учебники и справочники | Староверов И.Г. под ред., Богословский В.Н., Кокорин О.Я., Петров Л.В., российские вузовские пособия по ОВК |
| Оборудование и типовые данные | Каталоги российских производителей ПВУ/вентиляционного оборудования, например VERTRO, ВЕЗА, Арктос/Арктика, РОВЕН, Вентматик |
| Техническая реализация UI | Context7 и официальные docs Three.js/Dash только для кода, renderer lifecycle, clientside callbacks и performance guardrails |

## Prompt

```text
Ты работаешь в репозитории C:\My\Work\Diploma\ahu-simulator.

Цель: продолжить реализацию и документирование 3D-визуализации ПВУ/AHU в
русскоязычном инженерном контексте. Визуальная цель остается прежней:
- docs/assets/3d_visualization_final_reference_ru.png

Обязательное ограничение по источникам:
- Нормативы, литература, исходные данные, терминология, сценарии и пояснения
  должны быть отечественными или русскоязычными.
- Не использовать ASHRAE, CIBSE, REHVA, зарубежные учебники, зарубежные
  климатические базы и маркетинговые материалы иностранных производителей как
  первичную основу для диплома.
- Если иностранный источник нужен только для сравнения, явно помечай его как
  "не является базовым источником проекта" и не выводи из него расчетные
  параметры.
- Context7 разрешен только для технической документации библиотек Three.js/Dash:
  canvas sizing, renderer cleanup, OrbitControls, Dash assets/clientside
  callbacks/dcc.Store. Context7 не является источником инженерных норм ОВК.

Иерархия отечественных источников:
1. Нормативные документы РФ и официальные публикации:
   - Росстандарт / protect.gost.ru;
   - Минстрой России / minstroyrf.gov.ru;
   - Официальный портал правовой информации / publication.pravo.gov.ru;
   - Роспотребнадзор, если речь о санитарных требованиях.
2. Климатические и метеорологические данные:
   - СП 131.13330.2025 "Строительная климатология";
   - Росгидромет и ГГО им. А.И. Воейкова для климатических докладов и
     пояснительных материалов.
3. Учебная и справочная база:
   - Староверов И.Г. под ред. "Внутренние санитарно-технические устройства.
     Вентиляция и кондиционирование воздуха";
   - Богословский В.Н., Кокорин О.Я., Петров Л.В. "Кондиционирование воздуха и
     холодоснабжение";
   - российские вузовские пособия по отоплению, вентиляции и кондиционированию.
4. Отраслевая практика:
   - журнал АВОК;
   - материалы НЭБ/РГБ, ИСТИНА МГУ, eLIBRARY, КиберЛенинка;
   - каталоги российских производителей вентиляционного оборудования.

Обязательные документы для проверки перед изменениями:
- docs/35_3d_visualization_upgrade_plan.md
- docs/36_3d_visualization_course_map.md
- docs/37_3d_visualization_start_prompt.md
- docs/38_3d_visualization_ru_source_prompt.md
- docs/10_sources.md

Правила поиска:
- Для нормативов сначала ищи по официальным доменам:
  site:protect.gost.ru, site:minstroyrf.gov.ru, site:publication.pravo.gov.ru.
- Для климатических данных сначала ищи СП 131.13330.2025 и материалы
  Росгидромета.
- Для литературы сначала ищи НЭБ/РГБ, библиотеки вузов, eLIBRARY, КиберЛенинку,
  ИСТИНА МГУ и АВОК.
- Для оборудования используй российские каталоги и паспорта, явно фиксируя, что
  это типовые справочные значения производителя, а не норматив.
- Википедию, агрегаторы и случайные PDF использовать только как навигационные
  подсказки, не как финальный источник.

Терминология UI и документации:
- Используй русские инженерные термины: ПВУ, приточная установка, ОВК,
  воздуховод, воздушный клапан, фильтр, калорифер/нагреватель, вентилятор,
  расход воздуха, перепад давления, температура подачи, температура наружного
  воздуха, обслуживаемая зона, микроклимат, кратность воздухообмена.
- Если в коде уже используется AHU или английский идентификатор, не ломай API и
  component IDs, но в UI/документации показывай русское название.
- Не заявляй CFD или физически точное моделирование объемных потоков; формулируй
  как "инженерная визуализация расчетного состояния упрощенной модели".

Что делать при добавлении данных:
1. Укажи источник рядом с параметром или в docs/10_sources.md.
2. Разделяй нормативные значения, учебные формулы, климатические данные и
   типовые каталожные значения оборудования.
3. Для каждого нового сценария укажи регион/город РФ, расчетный сезон и источник
   наружных параметров.
4. Не подменяй отсутствующий отечественный источник зарубежным без отдельной
   пометки и обоснования.

Стартовая задача:
- Перепроверь docs/10_sources.md и при необходимости добавь раздел
  "Отечественная источниковая база 3D/ОВК".
- Для дальнейших фаз 3D используй этот prompt как фильтр качества источников.
- После изменений обновляй To-Do и acceptance matrix, а в финальном отчете
  отдельно пиши, какие отечественные источники использованы.
```

## Быстрый чек-лист

- [ ] Есть ссылка на официальный норматив или надежную отечественную публикацию.
- [ ] Источник относится к нужному типу данных: норма, климат, учебная формула,
      каталог, статья.
- [ ] Зарубежный источник не используется как первичная база диплома.
- [ ] Context7 используется только для кода и фронтенд-библиотек.
- [ ] Русские UI-подписи не противоречат расчетной модели.
