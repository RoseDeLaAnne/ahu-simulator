# Дорожная карта

## Вехи

| Веха | Цель | Результат | Критерий выхода |
| --- | --- | --- | --- |
| M1 | Зафиксировать предметную область | утвержденные параметры, сценарии, метрики | есть согласованный список входов и выходов |
| M2 | Описать модель | формулы, допущения, ограничения, режимы | есть документ модели и тестовые расчетные точки |
| M3 | Собрать ядро | Python-модуль расчета и CLI/smoke-проверка | базовый сценарий считается без UI |
| M4 | Поднять API | FastAPI-приложение с валидацией | `/docs` открывается, запросы валидируются |
| M5 | Поднять 2D UI | Dash-панель с SVG-мнемосхемой и трендами | пользователь меняет параметры и видит обновление узлов и потоков |
| M6 | Добавить сценарии и предупреждения | готовые режимы, alarms, статусы, привязка к визуальным узлам | сценарии воспроизводимы и читаемы на 2D-модели |
| M7 | Подготовить фундамент под 3D | единая модель визуальных сигналов, IDs и scene bindings | 3D можно подключить без переписывания расчетного ядра |
| M8 | Подтвердить качество | тесты, валидация, экспорт | есть автотесты и сравнение с эталонными точками |
| M9 | Подготовить демонстрацию | локальный запуск, документация, demo-script | проект можно показать офлайн без ручной настройки |

## Рекомендуемая последовательность

1. Сначала согласование физической модели.
2. Потом ядро и тесты.
3. Потом API.
4. Потом 2D UI и сценарии.
5. Затем фундамент под 3D-визуализацию.
6. В конце упаковка, экспорт, защита.

## Минимальный критический путь

- Согласование параметров.
- Реализация собственного расчета.
- Валидация хотя бы на нескольких режимах.
- API.
- 2D UI.
- визуальная схема сигналов под будущий 3D-слой.
- Тесты.
- Локальный запуск.

## Что можно отложить без ущерба для MVP

- OpenModelica.
- Полноценный 3D viewer с детальной геометрией.
- Расширенный архив сценариев.
- PDF-отчеты.
- Глубокий event log.
- Будущая SCADA/OPC-интеграция.

## Новый целевой пакет улучшений (2026-04-18)

| Веха | Улучшение | Цель | Критерий выхода |
| --- | --- | --- | --- |
| M10 | Расширение симуляции по времени | сделать интерактивный запуск: `Старт / Пауза / Сброс`, дискретные шаги времени и повторяемое обновление состояния | пользователь запускает и останавливает прогон без перезагрузки страницы, шаги времени отражаются на графиках и KPI |
| M11 | Отчетность по сценарию `PDF/CSV` | дать быстрый экспорт результата прогона в инженерный и табличный формат | экспорт доступен из UI и API, файлы сохраняются в `artifacts/exports/` с manifest |
| M12 | Сравнение прогонов `до/после` | упростить оценку эффекта изменений параметров | есть явный режим сравнения 2 прогонов с таблицей различий и графиком дельт |
| M13 | Цветные статусы `Норма/Риск/Авария` | сделать состояние установки читаемым с первого взгляда | единые правила статусов применяются в карточках, мнемосхеме, трендах и export |
| M14 | Пресеты сценариев `Зима/Лето/Пик` | ускорить демонстрацию и типовые проверки | пресеты доступны в UI и API, применяются в один клик и попадают в архив/экспорт |

### Рекомендуемая последовательность для реализации пакета M10-M14

1. Сначала `M10` и `M13`, чтобы получить живой и понятный интерфейс.
2. Затем `M14`, чтобы ускорить запуск типовых кейсов.
3. Затем `M11`, чтобы сразу формировать артефакты для обсуждения.
4. В конце `M12`, чтобы закрепить аналитический режим `до/после`.
