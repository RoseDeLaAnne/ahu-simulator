В dashboard добавлены встроенные `Defense Pack`, `Demo Readiness`, `Demo Bundle`, `Export Pack` и `Scenario Archive`, для проекта есть launch/preflight checklist, runtime snapshot, browser/WebGL profile, таблицы по стеку и модулям, заметки по использованию ИИ, локальный архив JSON-снимков, сборка `CSV/XLSX/PDF` export-наборов и реальная сборка offline-пакета с manifest, но финальный rehearsal всего demo-flow на целевом демо-ПК всё ещё не завершён.
- Фаза 9: завершена для post-MVP foundation текущей версии. Добавлены `Event Log`, live-блок `Control Modes`, оценка OpenModelica-адаптера закреплена в `docs/20_post_mvp.md`, а воспроизводимый контейнерный запуск подготовлен через `deploy/docker-compose.yml`.

- Phase A `Status Language Foundation` завершена: единый пользовательский язык
  статусов `Норма` / `Риск` / `Авария` берется из `StatusService`, технические

  `start/pause/reset/manual tick/auto tick`, speed control, horizon completion,

- Phase C `Reporting v2` завершена: отчетный контракт `scenario-report.v2`
  включает sections, chart specs, CSV/PDF/manifest artifacts, checksums/file
  metadata, preview, direct downloads и batch export.

  `До`/`После`, compatibility summary, interpretation/top deltas и
  `run-comparison.v2` export package.
- Phase E `User Presets v2` завершена: системные пресеты locked/read-only,
  пользовательские пресеты живут в runtime directory и поддерживают
  create/update/rename/delete/import/export через API и dashboard.
- Phase F `Documentation and Release Readiness` синхронизирует документы,
  `.omx` acceptance matrix и regression evidence; финальный browser walkthrough
  выполняется как smoke перед защитой на целевом окружении.
