# Функциональность проекта

## MVP-функциональность

- Интерактивная модель приточной вентиляционной установки с собственной логикой расчета.
- Главный экран с полноценной 2D-моделью установки на основе SVG-мнемосхемы: наружный воздух, фильтр, нагреватель, вентилятор, воздуховод, помещение, датчики, потоки и визуальные статусы узлов.
- Изменение входных параметров пользователем:
  - температура наружного воздуха;
  - расход воздуха;
  - уставка температуры притока;
  - эффективность рекуперации;
  - доступная мощность нагревателя;
  - степень загрязнения фильтра;
  - относительная скорость вентилятора;
  - температура помещения;
  - тепловая нагрузка помещения;
  - режим управления `auto/manual`.
- Пересчет состояния после изменения параметров.
- Отображение выходных показателей:
  - температура приточного воздуха;
  - температура в помещении;
  - фактический расход воздуха;
  - тепловая мощность;
  - суммарная мощность;
  - перепад давления на фильтре;
  - текущий режим: норма, риск, авария.
- Интерактивные графики и тренды.
- Готовые сценарии:
  - зима;
  - межсезонье;
  - снижение подачи воздуха;
  - загрязнение фильтра;
  - ручной режим.
- Блок сообщений и предупреждений.
- Адаптивное масштабирование 2D-модели без потери читаемости на разных размерах экрана.
- Подготовленная структура визуальных сигналов, чтобы те же статусы и значения можно было в следующем этапе подать в 3D-представление.
- REST API для запуска и получения результатов расчета.
- Автотесты для базовых, граничных и сценарных режимов.
- Локальный запуск без обязательных внешних сервисов.

## Усиливающая функциональность

- Отдельный 3D-режим визуализации установки на основе того же набора сигналов и состояний, что и 2D-модель.
- Переключение между 2D- и 3D-представлением без изменения расчетного ядра.
- Архив пользовательских сценариев.
- Экспорт результатов в `CSV`, `XLSX`, `PDF`.
- Отчетный пакет `scenario-report.v2` с preview, CSV/PDF/manifest и batch export.
- Сравнение прогонов `до/после` через runtime-снимки, interpretation summary и `run-comparison.v2` export.
- Пользовательские runtime-пресеты `scenario-preset.v2` без изменения системных preset-файлов.
- Журнал событий и изменений состояний.
- Ручной и автоматический режимы.
- Упрощенная логика автоматики и ПИД-регулирования.
- Теговая модель сигналов как задел под будущую SCADA/OPC-интеграцию.

## Нефункциональные требования

- Приложение должно работать локально и предсказуемо на демонстрационном ПК.
- Основной сценарий демонстрации не должен зависеть от интернета.
- Модель должна быть воспроизводимой: одинаковые входные параметры дают одинаковый результат.
- Интерфейс должен быть понятен без длительного обучения комиссии.
- Код модели должен быть отделен от UI.
- Визуальный слой должен быть отделен от расчетного слоя через явную модель визуальных сигналов.
- 2D-визуализация должна сохранять адаптивность и четкость при масштабировании.
- Закладываемый фундамент под 3D не должен делать WebGL обязательным для MVP-показа.
- Должна существовать минимальная автоматическая проверка расчета.

## Что считается готовым MVP

- Есть рабочий экран с параметрами и 2D-визуализацией состояния установки.
- Есть собственный расчетный модуль, а не заглушка.
- Есть минимум четыре сценария и набор предупреждений.
- Есть документированная схема визуальных сигналов и привязки узлов, пригодная для будущего 3D-слоя.
- Есть API-слой и тесты smoke-level.
- Есть воспроизводимый локальный запуск.

## Что не входит в MVP

- Полноценная интерактивная 3D-сцена с камерой, освещением и детальной геометрией оборудования.
- Подключение к реальному оборудованию.
- Работа с реальными конфиденциальными данными предприятия.
- Полноценная промышленная SCADA-система.
- Многопользовательский доступ и разграничение ролей.
- Производственный deployment вне демонстрационной задачи ВКР.
